# Issue #271 — fix(web): burger menu breaks UI layout on iOS/mobile viewport

## Status: APPROVED

## Summary
Pure CSS/JS layout bug in `landing-page/index.html`. No Swift app code changes. No module boundary implications.

## Root cause (from code inspection)
1. `.nav__cta` (Download Free button) is not hidden at `max-width: 768px` — it sits in the flex nav bar alongside the hamburger, causing overflow at 375px.
2. When `.nav__links.open` renders as `position: fixed; inset: 0; z-index: 101`, the nav bar is at z-index 100 (behind overlay). The hamburger stays visible at z-index 102 but the `.nav__cta` has no explicit z-index so it is occluded by the overlay — visually broken.
3. No body scroll lock when overlay is open (minor iOS usability gap).

## Architectural scope
- Single file: `landing-page/index.html`
- Isolated from all Swift app modules
- Deployed via Cloudflare Workers (`wrangler.toml`)
- Existing CSS structure (fixed overlay + z-index layering) is the right pattern; fix is at CSS property level, not structural

## Non-blocking caution
The Download CTA is absent from the mobile overlay (it's in the nav bar only). If it should appear in the overlay too, that's a two-line CSS/HTML change — not a structural concern.

## Resolved by
Code inspection of `landing-page/index.html` lines 100–165, 812–853, 895–913. No new seams, no coupling risk.
