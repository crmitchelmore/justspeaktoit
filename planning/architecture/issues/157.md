# Issue #157 — Add live capture health details to the HUD

**Status:** architecture-approved
**Date:** 2026-03-25

## Summary

Extend the macOS HUD to surface microphone permission, active input device, transcription provider, and a latency signal.

## Architecture decision (confirmed)

- `CaptureHealthSnapshot` as a plain value `struct` in `SpeakApp` (not `SpeakCore`) — all four data sources are macOS-only.
- Owned by `HUDManager`; nested in `HUDManager.Snapshot` or as a sibling `@Published var` — both are architecturally sound.
- `MainManager.setupBindings()` subscribes to `PermissionsManager.statuses`, `AudioInputDeviceManager`, and `AppSettings.liveTranscriptionModel` Combine publishers. No polling.
- Static `LatencyTier` from `ModelCatalog` for v1 — no runtime sampling. `LatencyBadge.swift` already renders it.
- `CaptureHealthSnapshot` must have value semantics: no closures, no `@ObservableObject`, no references to source managers.

## Blockers resolved

- Update trigger: resolved by maintainer's explicit Combine subscription list.
- Latency signal: confirmed static tier is sufficient for v1.
- Module boundary: confirmed SpeakApp-only.
