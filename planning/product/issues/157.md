# Issue #157 — Add live capture health details to the HUD

**Status**: Product-approved and product planning contribution made (2026-04-09)

## Summary
Adds mic permission, input device, provider, and latency-bucket to the existing macOS HUD so users can diagnose transcription failures without opening settings.

## Key facts verified
- HUD is macOS-only (confirmed in code: `#if os(macOS)`)
- All four data sources already exist: PermissionsManager, AudioInputDeviceManager, TranscriptionManager (provider registry), LatencyBadge
- No new managers or analytics needed
- HUDManager.Snapshot is the natural aggregation point; no CaptureHealth type exists yet
- Snapshot struct has phase, headline, subheadline, elapsed, liveText, etc — no health fields yet

## Product decisions (resolved 2026-04-09)
1. **Visibility**: Show in idle/ready state AND recording phases. Discovering problems before capture is more valuable than during. Skip health fields in terminal states (success/failure).
2. **Latency signal**: Reuse existing LatencyBadge tier (fast/medium/slow bucket). No per-session tracking.

## Scope guardrails
- Health display is read-only — no tappable shortcuts or inline permission grants
- No new settings to control health display visibility
- Exactly four data points (mic permission, device, provider, latency tier) — no additions
