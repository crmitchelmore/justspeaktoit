# Issue #157 — Add live capture health details to the HUD

**Status:** needs-quality (blocking)
**Date:** 2026-03-25

## Summary
Add a lightweight capture-health view to the existing HUD: mic permission state, input device name, active transcription provider, recent latency signal.

## Data sources (all available, no new infra needed)
- Mic permission: `PermissionsManager.statuses[.microphone]`
- Input device: `AudioInputDeviceManager`
- Provider: `SwitchingLiveTranscriber` / `AppSettings`
- Latency: `ModelCatalog.Option.latencyTier` (static) OR `TranscriptionResult.duration` (measured)

## Open blockers
1. **Ownership shape**: separate `@Published var captureHealth` on HUDManager, or a new `CaptureHealthMonitor`? HUDManager.Snapshot is a recording-phase state machine — health is independent of recording state.
2. **Latency signal**: static model tier vs. measured last-result duration — impacts ownership and test shape.
3. **Update triggers**: must be explicit event-driven (permissions change, device change, provider change) — no polling.

## Non-blocking notes
- `CaptureHealthSnapshot` should be a plain struct in `SpeakApp` (macOS-only, no reason to put in SpeakCore).
- Test surface: `SpeakAppTests` unit test with mock managers; no UI test needed.

## Approval
Not yet — awaiting clarification on the three blockers above.
