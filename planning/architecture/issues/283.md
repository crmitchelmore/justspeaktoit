# Issue #283 — fix(ios): add missing SpeakCore import

**Status:** architecture-approved
**Labels at review:** planning:needs-architecture, planning:design-approved, planning:reliability-approved; product/security/performance/quality still pending

## Summary
`SpeakiOSApp.swift:93` references `OpenClawClient.Conversation` (defined in `SpeakCore`) but only imports `SpeakiOSLib`. PR #282 was the attempted fix, but was closed without merging.

## Architecture stance
Approved. Fix is one line: `import SpeakCore` at top of `SpeakiOSApp/SpeakiOSApp.swift`. Explicit direct import is correct — app entry point declaring its own dependency, not relying on transitive exposure. Alternative (`@_exported import SpeakCore` in SpeakiOSLib) would implicitly widen SpeakiOSLib's public surface and couple all its consumers to SpeakCore's API. Single file, single module, no new seams.

## Resolved blockers
- Repo verification confirms `OpenClawClient` is in `Sources/SpeakCore/OpenClawClient.swift:9`
- Current `SpeakiOSApp.swift` missing the import (only SwiftUI, SpeakiOSLib, UIKit)
