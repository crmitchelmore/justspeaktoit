# PR #189 — fix(mac): prevent Data.subdata crash from non-zero startIndex

**Branch:** fix/data-subdata-crash
**Status:** not yet — blocked on missing linked planning issue

## What it does
- 4-line fix in `Sources/SpeakApp/TranscriptionManager.swift`
- Replaces `pendingPCMData.removeFirst(offset)` with `pendingPCMData = Data(pendingPCMData.dropFirst(offset))` in all 4 PCM drain loops
- Affects both `AssemblyAILiveController` and `ModulateLiveController`
- Root cause: `Data.removeFirst(_:)` shifts `startIndex` (non-zero), causing `subdata(in: 0..<3200)` to trap when `startIndex == 3200`

## Quality assessment
- Fix is technically correct: `Data(dropFirst(offset))` materialises a new value with startIndex == 0
- All 4 call sites addressed consistently — no asymmetric fix
- Change is minimal and surgical; no new abstractions or ownership confusion
- No regression test added for the Data.startIndex invariant
