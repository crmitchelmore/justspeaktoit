# Issue #270 — fix(ios): Apple live transcription clears text after speech pause

## Status: performance-approved

## Summary
Narrow bug fix confined to `Sources/SpeakiOS/Services/iOSLiveTranscriber.swift`. The fix patches `commitIfImplicitReset` threshold and the error-path data-loss in `handleRecognitionResult`.

## Performance assessment
- Hot path: `handleRecognitionResult` fires on every SFSpeechRecognizer partial result. The fix adds O(1) guard comparisons and string assignments — negligible.
- `restartRecognitionTask` startup gap is pre-existing. No new restarts introduced by the fix (error-path commit is a string assignment, not a restart).
- No new polling, timers, background tasks, or network calls.

## Approval rationale
- Fix is additive to existing committed-text pattern; no hot-path overhead change.
- Restart latency already part of the existing `isFinal` restart path.
- Non-blocking guardrail: if error-path fix includes a restart, verify audio tap buffer doesn't grow unboundedly during restart gap (same pattern as existing isFinal restart).
