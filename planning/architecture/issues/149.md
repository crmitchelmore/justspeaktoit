# Issue #149 — Portable planning-memory pattern after signed-commit correction

**Status:** architecture-approved
**Last updated:** 2026-03-25

## Summary
Workflow-health retest that escalated into a portability-design decision for planning memory under signed-commit rulesets.

## Verified facts
- `PUT /repos/{owner}/{repo}/contents/{path}` produced `verified: false`, `reason: unsigned` in this repo. Contents API writes are NOT signed-commit-safe here.
- `planning/*` branches are not blocked by required signed commits here, so the current topology works.
- `Docs/agentic-workflows.md` documents the two valid portability paths for strict repos.

## Architecture stance (corrected and final)
- Do **not** use Contents API as the signed-commit-safe default.
- **Portable default:** workflow commit signing — preserves `planning/*` branch topology across repos without topology changes.
- **Repo-local fallback:** branch exemption for `planning/*` when governance allows.
- Dedicated `{org}/planning-memory` repo only warranted when ≥3 code repos share a planning team.
- Docs now reflect this correctly; no application code changes in this issue.
