# Issue #157 — Add live capture health details to the HUD

**Status:** architecture-approved
**Date:** 2026-03-25

## Summary

Extend the macOS HUD to surface microphone permission, active input device, transcription provider, and a latency signal.

## Agreed plan (maintainer proposal, 2026-03-25)

1. `HUDManager` owns `CaptureHealthSnapshot` — distinct struct, stays in SpeakApp.
2. Static `LatencyTier` from `ModelCatalog`/`LatencyBadge` for v1 (no new measurement).
3. Update triggers: `PermissionsManager.statuses`, `AudioInputDeviceManager`, `AppSettings.liveTranscriptionModel` — Combine subscriptions only, no polling.
4. HUD shows categorical error labels only (no raw provider payloads).

## Architecture decision

- `CaptureHealthSnapshot` belongs in SpeakApp, not SpeakCore. All four sources are macOS-only.
- `MainManager` drives updates via existing `setupBindings()` pattern.
- `LatencyBadge.swift` already exists for rendering.
- No new coupling topology required.

## Resolved questions

- Update trigger: explicit Combine state-change publishers (resolved by maintainer 2026-03-25).
- Latency signal: static `LatencyTier` confirmed for v1.
