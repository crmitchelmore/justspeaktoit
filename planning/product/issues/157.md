# Issue #157 — Add live capture health details to the HUD

**Status**: Product-approved (phase-visibility tension resolved 2026-04-10)

## Summary
Adds mic permission, input device, provider, and latency-bucket to the existing macOS HUD so users can diagnose transcription failures without opening settings.

## Key facts verified
- HUD is macOS-only (confirmed in code: `#if os(macOS)`)
- HUD phases: `hidden`, `recording`, `transcribing`, `postProcessing`, `delivering`, `success(message:)`, `failure(message:)`. NO `idle` or `ready` phase exists.
- All four data sources already exist: PermissionsManager, AudioInputDeviceManager, TranscriptionManager (provider registry), LatencyBadge
- No new managers or analytics needed
- HUDManager.Snapshot is the natural aggregation point

## Product decisions (final)
1. **Visibility**: Show health ONLY during `.recording` and `.failure` phases. Product's idle/ready suggestion in Comment 22 was based on a non-existent HUD state — retracted. Design spec confirmed correct.
2. **Latency signal**: Reuse existing LatencyBadge tier. No per-session tracking.

## Scope guardrails
- Health display is read-only — no tappable shortcuts or inline permission grants
- No new settings to control health display visibility
- Exactly four data points (mic permission, device, provider, latency tier)
- Provider/error fields: fixed categorical labels only (no raw API payloads)
