# Issue #270 — fix(ios): Apple live transcription clears text after speech pause

**Status**: Product-approved
**Date**: 2026-04-08
**Author**: bravostation (collaborator)

## Summary
iOS bug: text accumulated during a live Apple Speech session disappears after a pause in speech, forcing the user to re-speak.

## Product Assessment
- Platform: iOS ✓
- User moment: live transcription capture ✓
- Engine: Apple SFSpeechRecognizer ✓
- Clear reproduction steps ✓
- Root cause hypothesis reasonable and specific ✓

## Code evidence
`Sources/SpeakiOS/Services/iOSLiveTranscriber.swift` already contains partial mitigation:
- `committedText` accumulator
- `commitIfImplicitReset()` heuristic (triggers if currentText < half of lastFormattedString)
- `restartRecognitionTask()` on mid-session isFinal

Bug is still reproducible → the existing heuristic has edge cases (e.g. silence-timeout-driven task death, or short text strings < 10 chars not triggering the guard). Fix belongs in this file.

## Scope boundary
- iOS only; macOS uses a different transcription stack
- Fix should not affect `stop()` semantics or `onFinalResult` callback
- Non-blocking note to engineer: verify fix works for both on-device and server-based recognition paths

## Decision
Approved. `planning:product-approved` added, `planning:needs-product` removed.
