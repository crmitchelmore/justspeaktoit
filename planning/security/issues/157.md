# Issue #157 — Add live capture health details to the HUD

## Summary
Adds microphone permission state, input device name, active transcription provider, and a latency bucket to the existing macOS HUD.

## Security stance
**Approved** — `planning:security-approved` label set; approval comment at #4122668274.

## Rationale
- All displayed fields are categorical (permission bool, device name from system API, provider label, latency bucket).
- No new auth surface, no new data collection, no new analytics vendor.
- `CaptureHealthSnapshot` as a distinct value type enforces categorical fields at the type boundary.
- Health type stays in `SpeakApp`, not `SpeakCore` — confirms no cross-platform data leakage risk.
- Explicit refresh triggers only (permissions, device selection, model changes) — no background polling.

## Guardrails to enforce at implementation
- Provider/error labels must be categorical strings, not raw API error bodies or HTTP response text.
- `finishFailure(message:)` already renders message strings directly in HUD — implementation must not pass raw provider API error strings through health state.
- Health aggregator state must not be logged or sent to Sentry in ways that capture raw API responses.
