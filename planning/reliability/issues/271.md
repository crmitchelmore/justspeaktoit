# Issue #271 — fix(web): burger menu breaks UI layout on iOS/mobile viewport

**Status**: Approved — 2026-04-08

## Summary
Landing page visual regressions on mobile viewport:
1. Burger/hamburger menu breaks UI when opened
2. Download button in top nav looks bad at mobile sizes
Fix scope: `landing-page/` only (Cloudflare Pages static site).

## Reliability assessment

### Blast radius
Minimal: landing page HTML/CSS/JS only. No macOS app, no iOS app, no API, no database, no credentials. Any failure affects only the marketing website.

### Deployment pipeline
- `deploy-landing-page.yml`: triggers on push to `main` paths `landing-page/**` 
- Direct deploy to Cloudflare Pages via `wrangler-action@v3`
- **No PR preview deployment step** — changes go live on main merge
- No build step (static site, `echo 'Static site - no build needed'`)
- Rollback: Cloudflare Pages dashboard — instant rollback to prior deployment

### Failure modes
1. CSS change regressions desktop viewport — mitigated by code review; non-blocking caution
2. JS menu fix breaks non-iOS browsers — test at multiple viewports before merge
3. Deploy failure — CI gives clear success/failure; Cloudflare Pages rollback available
4. No staging/preview gate before production

### Approval rationale
Static site, instant rollback, narrow blast radius. No app code or infrastructure at risk. Direct-to-prod deploy on merge is acceptable for a landing page with Cloudflare's rollback.

## Guardrails (non-blocking)
1. Test fix at desktop viewport before merging — avoid nav regression
2. Rollback path: Cloudflare Pages dashboard → previous deployment
3. Note absence of PR preview deployment (acceptable risk; could be added as enhancement)
