# Issue #157 — Add live capture health details to the HUD

**Status:** architecture-approved (re-approved 2026-04-10 after /doit cycle)

## Summary
Extend macOS HUD with mic permission, input device, transcription provider, and latency signal.

## Architecture boundaries
- `CaptureHealthSnapshot` plain struct in `SpeakApp` (no SpeakCore)
- Updated via `HUDManager.updateCaptureHealth(_:)` driven by `MainManager` bindings
- Three update triggers: `PermissionsManager.statuses`, `AudioInputDeviceManager`, `AppSettings.liveTranscriptionModel`
- Static `LatencyTier` for v1; no back-references to source managers
- Reuse existing `LatencyBadge.swift`

## Cross-role note
Product vs Design tension on which phases to show health panel — view-layer only, no structural implication. PR must record chosen phase set.
