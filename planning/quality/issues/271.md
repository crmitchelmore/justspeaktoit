# Issue #271 — fix(web): burger menu breaks UI layout on iOS/mobile viewport

## Stance
Approved.

## Context
Pure landing-page HTML/CSS/JS fix. Single file: `landing-page/index.html`.
No automated tests for the landing page — manual browser verification is the appropriate bar.

## Root cause (verified in code)
1. `.nav__cta` has no `display: none` at `max-width: 768px` — download button stays visible beside hamburger.
2. Hamburger JS toggle only flips `.open` class; no `document.body.style.overflow` toggle — iOS Safari scroll-through.
3. No `aria-expanded` on `.nav__hamburger` (minor, non-blocking).

## Guardrails for implementation
- Add `display: none` to `.nav__cta` inside the existing `@media (max-width: 768px)` block.
- Toggle `overflow: hidden` on `body` in hamburger click handler; restore in both close paths.
- Single file scope — no new abstractions needed.
