# PR #298 — fix(mac): disable HUD glass effect on macOS 26

**State:** Merged 2026-04-10T22:49:46Z
**Branch:** fix/mac-disable-glass-26x
**Author:** crmitchelmore

## Summary
- `canUseGlassEffect()` simplified to `version.majorVersion >= 27` — disables glass on all macOS 26 builds
- `isGlassEffectEnabled: Bool` added as a `static let` cache — avoids repeated `ProcessInfo` reads on every HUD render
- `HUDView.hudShell()` now reads `HUDPlatformWorkarounds.isGlassEffectEnabled` (static let) instead of a computed var
- Tests updated to assert glass disabled for 26.x, enabled only on 27+

## Performance stance
The `static let` cache is the correct fix for a hot path — HUD renders frequently during active recording, and `ProcessInfo.processInfo.operatingSystemVersion` reads are non-trivial. Caching this at startup is cheap and correct. No linked planning issue was found in the PR body.
