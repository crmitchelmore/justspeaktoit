# Issue #157 — Add live capture health details to the HUD

## Status
`planning:needs-performance` — initial review, no other roles have commented yet.

## Summary
Add a lightweight capture-health view to the HUD: mic permission, input device, provider, and a recent-latency signal.

## Verified repo facts
- HUDManager is a push-based state machine with `@Published` properties; updates come from callers (`updateAudioLevel`, `updateLiveTranscription`, etc.), not a polling loop.
- An internal `Timer` runs on `RunLoop.main` for step-duration display — additional properties can piggyback on this pattern.
- `TranscriptionManager` tracks session duration and segment timing but no latency bucket is currently published to HUDManager.
- `PermissionsManager` already exists; mic permission state can be observed rather than polled.

## Open performance questions
1. **Latency signal implementation**: How is "recent transcription latency" computed and stored? A single last-value vs. rolling average, and at what update frequency?
2. **Update rate cap**: Audio level updates are high-frequency; new health fields must be rate-limited or only updated on state transitions.
3. **Device enumeration**: AVAudioEngine/CoreAudio device enumeration is non-trivial — must be notification/KVO-driven, not polled.

## Approval
Not yet approved. Waiting for latency signal design and update-rate confirmation.
