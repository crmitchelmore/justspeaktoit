# Issue #174 — Port final career-framework agentic workflow pattern

**Status**: approved

## Summary
Workflow/automation cleanup on a private repo. Ports the career-framework planning + PR-review pattern. Removes the general-purpose triage bot in favour of Product-on-open validation. Adds named personas to role memory.

## Security landscape
- Private repo: collaborator-only access — spam-detection loss from removing `issue-triage.lock.yml` is not material.
- Named `persona.md` memory files: same isolation pattern as existing memory — stored in `planning/*` branch, written only by agent. Low risk.
- PR plan-review restricted to PR threads only: scope-narrowing improvement.
- PR reconcile: label-state-only decision (confirmed by maintainer). No comment parsing for merge-readiness.
- Permissions: `contents: read`, `issues: write`, `pull-requests: write` — least-privilege.
- `issue_comment` trigger is idempotent nudge only; duplicate triggers normalise labels without creating a second source of truth.
- Atomic removal: no double-fire window; triage files and new product-validation path land in the same PR.
