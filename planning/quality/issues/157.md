# Issue #157 — Add live capture health details to the HUD

**Status:** needs-quality (blocking)
**Date:** 2026-03-25 (second run)

## Summary
Add a lightweight capture-health view to the existing HUD: mic permission state, input device name, active transcription provider, recent latency signal.

## Code facts verified
- `HUDManager.Snapshot` (Sources/SpeakApp/HUDManager.swift) is a recording-phase state machine with Phase enum (.hidden, .recording, .transcribing, etc.) — health data is independent and must not go in Snapshot
- `PermissionsManager.statuses: [PermissionType: PermissionStatus]` — in SpeakApp, @Published
- `AudioInputDeviceManager.devices`, `selectedDeviceUID`, `activeDeviceUID` — in SpeakApp, @Published  
- `SwitchingLiveTranscriber` — in SpeakApp (TranscriptionManager.swift)
- `TranscriptionResult.duration: TimeInterval` — in SpeakCore/LLMProtocols.swift (measured latency)
- `ModelCatalog.Option.latencyTier: LatencyTier` — in SpeakCore/ModelCatalog.swift (static tier)
- `CaptureHealthSnapshot` and `CaptureHealthMonitor` — do NOT exist, greenfield

## Open blockers
1. **Ownership shape**: separate `@Published var captureHealth` on HUDManager, or a new `CaptureHealthMonitor`? Issue body leaves open.
2. **Latency signal**: static model tier vs. measured last-result duration — unresolved in issue.
3. **Update triggers**: must be explicit event-driven — issue says "no polling" but doesn't specify the observer wiring.

## Non-blocking notes
- `CaptureHealthSnapshot` belongs in `SpeakApp` (macOS-only, no reason in SpeakCore).
- Test surface: `SpeakAppTests` unit tests with mock managers.
