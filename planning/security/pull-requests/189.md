# PR #189 — fix(mac): prevent Data.subdata crash from non-zero startIndex

**Status:** needs-security (blocked on missing plan issue)
**Date opened:** 2026-03-26

## What it does
4 identical line changes in `Sources/SpeakApp/TranscriptionManager.swift`.  
Replaces `pendingPCMData.removeFirst(offset)` → `pendingPCMData = Data(pendingPCMData.dropFirst(offset))`.  
Affects both AssemblyAILiveController and ModulateLiveController audio chunking loops.

## Security assessment
- Pure crash-fix. No trust boundary changes.
- No auth/authz, secrets, permissions, or data-exfiltration paths touched.
- Audio PCM buffer stays in-process; the fix just ensures zero-based Data after draining.
- Fix is correct: `Data(dropFirst(offset))` produces a fresh zero-based copy.
- Blocking an out-of-bounds trap is security-positive (prevents availability disruption).
