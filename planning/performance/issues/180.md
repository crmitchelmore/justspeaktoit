# Issue #180 — Ignore closed-PR comments in plan-review workflows

## Status: approved

## Summary
Pure CI/workflow guard change. Adds `github.event.issue.state == 'open'` check to PR plan-review workflow `issue_comment` triggers to prevent spurious runs on already-closed PRs.

## Performance stance
No app runtime code. No hot path. Actually *reduces* unnecessary CI workflow fan-out — positive from operational cost perspective. Consistent pattern with issues #174 and #176.

## Approved: 2026-03-25
No performance concerns. Guardrail: change must not affect `pull_request` and `workflow_dispatch` trigger paths.
