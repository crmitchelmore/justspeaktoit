# PR #189 — fix(mac): prevent Data.subdata crash from non-zero startIndex

## Summary
Crash fix: `Data.removeFirst(_:)` shifts `startIndex` instead of reindexing to zero. `subdata(in: 0..<3200)` then crashes with `EXC_BREAKPOINT` when startIndex != 0. Fix replaces all four `pendingPCMData.removeFirst(offset)` calls with `Data(pendingPCMData.dropFirst(offset))`.

Affects both Modulate and AssemblyAI live streaming audio processors.
