# Issue #271 — fix(web): burger menu breaks UI layout on iOS/mobile viewport

**Status**: Product approved (2026-04-10) — all roles approved → planning:ready-for-dev
**Author**: bravostation (COLLABORATOR)
**Surface**: landing-page/index.html (static marketing site)

## Problem
Two bugs on mobile viewport (≤768px):
1. Hamburger overlay clamped to nav height on iOS Safari (backdrop-filter containment bug)
2. `nav__cta` (Download Free) button visible in nav bar at 375px — overflows layout

## Code-confirmed diagnosis (Riley + Morgan + Casey aligned)
- `.nav__links.open` is `position: fixed` inside `.nav` which has `-webkit-backdrop-filter: blur(8px)` — creates containing block on iOS Safari, clips overlay to ~70px nav height
- `.nav__cta` sibling of nav links, no mobile hide rule — layout overflow source
- No body scroll lock; no `aria-expanded` on hamburger

## Implementation guardrails (Riley's blocking items, all accepted)
1. Move mobile overlay to sibling of `<nav>` — escapes -webkit-backdrop-filter containment
2. Add `display: none` for `.nav__cta` at ≤768px
3. Toggle `aria-expanded` on `.nav__hamburger` (WCAG 4.1.2)
4. Add Escape key listener to close menu (WCAG 2.1.1)

## Product assessment
- User: website visitors on mobile/iOS — primary conversion surface
- Scope: landing-page only, no app code changes
- Success: clean overlay open, correctly styled Download button at all viewport sizes, tested at 375px iOS Safari + Chrome responsive mode
