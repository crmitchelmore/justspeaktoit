# Issue #162 — Add plan-linked PR review after issue planning

## Status
**Approved** — all five roles approved; `planning:ready-for-dev` set.

## Summary
Extends the planning workflow to add a PR review stage. Implementation PRs must link their approved planning issue via `Plan issue: #<n>`. All five planning roles review the PR against the issue plan, the diff, and verification evidence.

## Product assessment
Clear user value: closes the gap between "approved plan" and "reviewed implementation". Well-scoped, no user-facing UX to worry about. Requirements are specific and testable. Non-goals are implicit (no feature code, no runtime changes).

## Guardrails noted at approval
- Manual dispatches from feature branches must not accidentally reconcile unrelated open PRs.
- PR template must make `Plan issue:` field obviously required (not a placeholder that looks valid).
- The flow should be documented for portability.

## Key resolved blockers
None noted — issue approved in initial state with no recorded Product blockers.
