# PR #191 — fix(agentic-workflows): disable failure issue creation on all workflows

**Date opened:** 2026-03-26
**Branch:** chore/silence-failure-issues
**Status:** BLOCKED (missing plan issue linkage)

## What it does
Sets `GH_AW_FAILURE_REPORT_AS_ISSUE: "false"` in all 31 workflow lock.yml files.
Adds `report-failure-as-issue: false` in corresponding .md source files.
Prevents agentic workflow failures from auto-creating `[aw] ... failed` issues.
Failures remain visible in the Actions tab.

## Security assessment
- Pure noise-reduction config change. Zero trust boundary impact.
- No permissions, auth/authz, secrets, or data-handling changes.
- Slightly reduces observability (no issue auto-created on failure) but Actions tab still shows failures.
- Workflow permissions and fork guards are unchanged throughout.
- All 31 workflows affected consistently — no partial / split rollout risk.
- Security-neutral to positive: less automated issue noise, same failure signal via Actions.
