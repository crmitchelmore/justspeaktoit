# Issue #270 — fix(ios): Apple live transcription clears text after speech pause

## Status
Architecture: **approved** (re-approved 2026-04-09 after second /doit reset labels; identical plan to 2026-04-08 approval)

## Summary
Bug in `iOSLiveTranscriber.swift`: transcribed text is cleared when Apple's SFSpeechRecognizer times out on silence. The codebase already has a committed-text accumulation pattern (`committedText`, `commitIfImplicitReset`, `restartRecognitionTask`) but two gaps allow text loss.

## Structural analysis
- Fix is correctly scoped to `Sources/SpeakiOS/Services/iOSLiveTranscriber.swift` — iOS-only, no cross-module changes.
- Two gap areas:
  1. **Error path** — `handleRecognitionResult` returns without committing `lastFormattedString` when a non-shutdown error arrives.
  2. **Heuristic threshold** — `commitIfImplicitReset` requires `lastFormattedString.count >= 10`; short utterances are not protected.
- No new seams, no interface changes.

## Guardrails (non-blocking)
- Commit must happen before any early return in the error path (unless `isShuttingDownRecognitionTask` is true — intentional cancel).
- New log statements must not log transcription content — char counts only.
- If error-path commit triggers a restart, verify audio tap buffer doesn't grow unboundedly.
