# Issue #271 — fix(web): burger menu breaks UI layout on iOS/mobile viewport

**Status**: Product approved (2026-04-08)
**Author**: bravostation (COLLABORATOR)
**Surface**: landing-page/index.html (static marketing site)

## Problem
Two bugs on mobile viewport (≤768px):
1. Hamburger menu opening breaks/disrupts UI layout
2. `nav__cta` (Download Free) button looks visually broken in mobile nav bar

## Code-confirmed diagnosis
- `nav__cta` is a sibling of `nav__links` in the HTML; no mobile-specific hide/reposition style exists
- Hamburger uses `position: fixed; inset: 0` overlay — z-index stack may conflict
- Single file scope: `landing-page/index.html`

## Product assessment
- User: website visitors on mobile/iOS — primary conversion surface
- Scope: landing-page only, no app code changes
- Success: clean overlay open, correctly styled Download button at all viewport sizes
- Approved: clear bug, real user pain, bounded scope, testable outcome
