# Issue #162 — Add plan-linked PR review after issue planning

## Scope
Pure workflow/tooling infrastructure. No app code changes. Extends the existing `gh-aw` agent system with a PR review stage parallel to issue planning.

## Architecture stance: APPROVED

### Structural fit
- New `pr-review-*.md` + compiled `.lock.yml` files mirror the existing `issue-planning-*.md` pattern exactly.
- A new `pr-review-reconcile.yml` mirrors `issue-planning-reconcile.yml` for the ready-to-merge gate.
- `Plan issue: #...` in PR body is the explicit, simple coupling point between issue plan and PR review.

### Guardrails noted (non-blocking)
- Label namespace must be `pr-review:*` (not `planning:*`) to avoid collision with issue labels.
- Manual dispatch scoping: the reconcile workflow should require an explicit PR number on `workflow_dispatch` to avoid reconciling unrelated PRs.
- All five PR review agents must parse `Plan issue:` consistently — centralise or document the regex.

### Resolved blockers
- Prior concern (PR #161 review): no linked plan issue. Issue #162 IS the plan issue. Resolved.
