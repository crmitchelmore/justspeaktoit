# Issue #157 — Add live capture health details to the HUD

**Status**: Product-approved (initial review, 2026-03-25)

## Summary
Adds mic permission, input device, provider, and latency-bucket to the existing macOS HUD so users can diagnose transcription failures without opening settings.

## Key facts verified
- HUD is macOS-only (confirmed in code: `#if os(macOS)`)
- All four data sources already exist: PermissionsManager, AudioInputDeviceManager, TranscriptionManager (provider registry), LatencyBadge
- No new managers or analytics needed
- HUDManager.Snapshot is the natural aggregation point; no CaptureHealth type exists yet

## Open product decisions (to resolve in implementation)
1. When is health info visible: embedded in recording phases only, or also surfaced in an idle/ready state?
2. Latency signal: use existing LatencyBadge tier or track per-session latency bucket separately?

## Approval rationale
Clear user problem, named surface (macOS HUD), specific data points, tight constraints, existing data sources. Low risk of scope creep given constraints in the issue body.
