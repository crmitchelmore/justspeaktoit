# Issue #157 — Add live capture health details to the HUD

## Summary
Adds microphone permission state, input device name, active transcription provider, and a latency bucket to the existing macOS HUD.

## Security stance
**Approved** (initial review, no prior planning team comments).

## Rationale
- All displayed fields are categorical (permission bool, device name from system API, provider label, latency bucket).
- No new auth surface, no new data collection, no new analytics vendor.
- Existing HUDManager already surfaces error `subheadline` — the new health fields follow the same pattern.

## Non-blocking guardrails noted
- Provider/error labels in the health view must be categorical strings, not raw API error bodies or HTTP response text.
- `finishFailure(message:)` already renders the message string directly in HUD — implementation must not pass raw provider API error strings through the health state.
- Health aggregator state must not be logged or sent to Sentry in a way that inadvertently captures raw API responses.
