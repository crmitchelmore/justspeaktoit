# Issue #271 — fix(web): burger menu breaks UI layout on iOS/mobile viewport

## Stance
Approved with guardrails.

## Context
Pure landing-page HTML/CSS/JS fix. Single file: `landing-page/index.html`.
No automated tests for the landing page — manual browser verification is the appropriate bar.

## Root cause (from code inspection)
1. `.nav__cta` (Download Free button) has no `display: none` at `max-width: 768px`, so it stays 
   visible beside the hamburger icon on narrow viewports. This is the "download button looks bad" bug.
2. The mobile overlay (`position: fixed; inset: 0`) has no body scroll lock — 
   iOS Safari allows scroll-through. This likely contributes to the "UI breaks" symptom.
3. The hamburger JS toggle doesn't manage `aria-expanded` (minor, non-blocking).

## Guardrails for implementation
- Hide `.nav__cta` on mobile (or include it in the overlay) — non-negotiable
- Add `document.body.style.overflow = 'hidden'/'auto'` toggle in the hamburger JS handler
- Verify in Chrome responsive mode at ≤375px AND iOS Safari (or Simulator)
